Recomendacoes

TC-PERF-S6-002 — Rate Limit da Open Trivia DB [ALTA]

A API publica open.trivia.com aplica rate limiting severo sob carga concorrente. Com 5 VUs simultaneos, 90.59% das requisicoes falharam (response_code=5). Ajustar o teste para no maximo 1-2 VUs com sleep de 2-3s entre iteracoes, ou usar um endpoint interno de trivia sem limitacao para testes de carga.

TC-VISUAL-S6-001 e TC-VISUAL-S6-002 — Baselines criados, validacao manual pendente [MEDIA]

Esta foi a primeira execucao dos testes visuais. Os baselines de webaim-contrast.png e pokeapi-home.png foram criados e devem ser revisados manualmente pela equipe antes de se tornarem referencia. Executar novamente para confirmar estabilidade.

TC-SEC-S6-001 e TC-SEC-S6-002 — Headers de seguranca opcionais ausentes [BAIXA]

As APIs Advice Slip e Open Trivia nao implementam Strict-Transport-Security, Content-Security-Policy e X-Frame-Options. Como sao APIs publicas de terceiros fora do controle do time, documentar como limitacao conhecida e excluir de futuras suites de seguranca ou ajustar os criterios de falha.
