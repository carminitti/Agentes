💡 Possíveis Melhorias

Recomendações

Ações prioritárias baseadas nas falhas, avisos e padrões encontrados na execução da Suite 3.

1

Migrar testes de API do Reqres.in para nova versão autenticada ou API alternativa

TC-API-S3-001 e TC-API-S3-002 falharam com 401 pois o Reqres.in passou a exigir x-api-key. Obter uma API key em app.reqres.in/api-keys e adicionar o header nas requisições, ou migrar para JSONPlaceholder (jsonplaceholder.typicode.com) que continua público. O endpoint /app/* do Reqres.in continua funcionando para sessões de usuário.

2

Corrigir violações de contraste de cor (WCAG 2.1 AA) — TC-A11Y-S3-001 e TC-A11Y-S3-002

Ambos os sites falharam com color-contrast (serious). Para practiceautomation.com: escurecer a cor dos botões do cookie banner de #2aa527 para pelo menos #1e8c1b (4.5:1). Para quotes.toscrape.com: ajustar a cor dos links de #4582ec para pelo menos #1a5bbf e adicionar sublinhado ao link "Zyte" no rodapé. Estes são sites de terceiros — marcar como known_environment_failure se não houver controle sobre eles.

3

Fornecer strings de conexão reais para os testes de banco — TC-DB-S3-001 e TC-DB-S3-002

Os testes de banco executaram em modo simulated_fallback (SQLite :memory:) pois os hosts MySQL e PostgreSQL especificados nos steps são inacessíveis (db.qa-suite3.com) ou placeholder (ep-placeholder.us-east-1.aws.neon.tech). As verificações de FK passaram com aviso pois o SQLite exige PRAGMA foreign_keys = ON explicitamente. Validar contra o banco real antes do deploy para garantir que as constraints de FK e CHECK estão devidamente configuradas no DDL de produção.

4

Validar baselines visuais criados na primeira execução — TC-VISUAL-S3-002

O baseline de quotes-love-tag.png foi criado na primeira execução em suite_api_browser_k6_visual_axe_zap_db_20260511_100754/visual/baselines/. Revisar visualmente o screenshot antes de usá-lo como referência oficial. Executar o teste novamente para confirmar que a comparação funciona corretamente. TC-VISUAL-S3-001 passou com 0% de diferença após criação do baseline.

5

Adicionar headers de segurança nas APIs de demonstração utilizadas para testes

TC-SEC-S3-001 reportou ausência de X-Content-Type-Options, X-Frame-Options, Strict-Transport-Security e Content-Security-Policy na FakeStore API, além de CORS aberto (*). Como é uma API pública de teste, o resultado foi classificado como warning (não bloqueante). Ao migrar para ambiente de staging ou produção próprio, garantir que esses headers estejam configurados no servidor/proxy.
